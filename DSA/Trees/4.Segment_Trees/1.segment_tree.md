# Segment Tree
A segment tree is a binary tree used for storing intervals or segments. It allows querying which of the stored segments contain a given point efficiently.

# When to use?
Segment trees are useful for performing range queries (such as sum, max, min, average, product) and updates on an array efficiently.

# Algorithm
## Build:
Start from the root and recursively build the left and right subtrees.
Combine the results of the left and right subtrees to form the parent node.

## Update at index:
If the current node's range is completely within the update range, add the value to the node.
If the current node's range is partially within the update range, propagate the update to the child nodes.
Recalculate the value of the current node based on its updated children.

## Query for range:
If the current node's range is completely within the query range, return the node's value.
If the current node's range is partially within the query range, query the child nodes.
Combine the results of the child nodes to get the final result.

# Range Addition:

## Build:
Start from the root and recursively build the left and right subtrees.
Combine the results of the left and right subtrees to form the parent node.
## Update:
If the current node's range is completely within the update range, add the value to the node.

Recalculate the value of the current node based on its updated children.
## Query:
If the current node's range is completely within the query range, return the node's value.
If the current node's range is partially within the query range, query the child nodes.
Combine the results of the child nodes to get the final result.


# Additional Information
## Lazy Propagation:
To optimize range updates, lazy propagation can be used. This technique delays updates to segments
until necessary, reducing the number of operations.

## Time Complexity:
The time complexity for building the segment tree is \(O(N)\), and
for both update and query operations, it is \(O(\log N)\).
