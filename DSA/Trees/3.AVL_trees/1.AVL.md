# Problem in BST:
if the incoming values are sorted, then the tree becomes unbalanced.
To fix this we need to balance the tree

# AVL
One of the algorithm to balance the tree

# How to balance?
we know, how to know if the tree is balanced or not from previous topics using heights
Now we have to understand tree Rotation
## Tree Rotation
A method to re arrange tree nodes such that BST is not disturbed
consider p = parent , c = child t1, t2, t3 are sub tress

![tree_rotation](https://github.com/noor-kallur/CodeClassRoom/blob/master/DSA/Trees/3.AVL_trees/2.tree_rotation.png)

# Algorithm
1. Insert normally node n
2. start from node n(lastly inserted) and find the node that makes the tree unbalanced (bottom up)
3. Using one the FOUR rules to rotate

## FOUR Rules
Goal is to make a trianlge of p , c and g
