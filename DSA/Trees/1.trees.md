## Types of Binary Tree

### 1. Complete Binary Tree
    A complete binary tree is a type of binary tree where all levels, except possibly the last,
    are fully filled, and all nodes are as far left as possible.

### 2. Full Binary Tree / Strict Binary Tree
    Each node has either 0 or 2 childern
    
    use cases : compression(huff man coding)
                segment trees

### 3. Perfect Binary Tree
    All leveles are full

### 4. Height Balanced Binary Tree
    A height-balanced binary tree is a type of binary tree where the difference in height between the
    left and right subtrees of any node is at most one.
    This helps keep the tree balanced, making operations like searching, inserting, and deleting efficient.
    
Average Height is $O(\log(n))$

### 5. Skewed Binary Tree
    Every node only has one child
Average height is $O(n)$

### 6. Orderd Binary Tree
    The nodes are arranged in an ordered manner to facilitate efficient searching, insertion, and deletion operations

    Eg: Binary search Tree

## Properties:
Total nodes in Perfect Binary Tree $2^{h+1} - 1$

Lead nodes in Perfect Binary Tree $2^h$

Number of Internal nodes(Total - Leaves)  in Perfect Binary Tree 

$$
2^{h+1} - 1 - (2^h) = 2^h - 1
$$

if N = No of leaves then the number of Levels are in Perfect Binary Tree

$$
\log(N) +1
$$

if N = No of leaves then the number of Levels are in Perfect Binary Tree

$$
\log(N +1)
$$

In a Strict Binary Tree:
    Let T = Total nodes Ln = Leaf Nodes In = Internal Nodes
    T = In + Ln
    If you observe carefully, Please draw this out
    Ln = In + 1 

In a non-empty binary tree,
Number of leaf nodes = 1 + Number of internal nodes with 2 children(inlcudes root)

$$
Ln = 1 + In(with 2 children)
$$
