# Subsets: Exploring Two Approaches

## ✅ Subsets1: Binary Decision Tree Approach

### Logic:
At each index `i`, you make a binary choice:
1. **Include** `nums[i]`
2. **Exclude** `nums[i]`

This forms a binary tree of decisions, with `2^n` leaves (where `n` is the length of `nums`).

### Code:
```python
def subsets1(nums):
    def backtrack(index, path):
        if index == len(nums):
            result.append(path[:])
            return
        # Include nums[index]
        path.append(nums[index])
        backtrack(index + 1, path)
        path.pop()
        # Exclude nums[index]
        backtrack(index + 1, path)

    result = []
    backtrack(0, [])
    return result
```

### Pros:
- Very intuitive.
- Explicitly shows the include/exclude logic.
- Always generates all subsets in lexicographic order.

## ✅ Subsets2: Iterative Expansion Approach

[Method visualisation](https://github.com/noor-kallur/CodeClassRoom/blob/master/DSA/Recursion/Backtracking/Basics/Combinations/78.subsets_method_2.png)
### Logic:
At each recursive level:
1. Add the current subset to the result.
2. Loop through the remaining elements and recursively build subsets.

### Code:
```python
def subsets2(nums):
    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()

    result = []
    backtrack(0, [])
    return result
```

### Pros:
- More compact.
- Easier to adapt for problems like combinations of size `k`.
- Slightly more efficient in practice due to fewer recursive calls.

---

## 🧠 Efficiency Comparison

| Feature            | Subsets1            | Subsets2            |
|---------------------|---------------------|---------------------|
| **Time Complexity** | `O(2^n)`           | `O(2^n)`           |
| **Space Complexity**| `O(n)` (recursion stack) | `O(n)` (recursion stack) |
| **Recursion Calls** | Exactly `2^n`      | Fewer due to loop pruning |
| **Flexibility**     | Good for binary decisions | Better for combinations |
| **Readability**     | Very clear         | Slightly more abstract |

---

## 🏆 Verdict
- Use **Subsets1** when you want to clearly model binary choices (include/exclude).
- Use **Subsets2** when you want a more compact and flexible approach, especially for problems like combinations or permutations.
